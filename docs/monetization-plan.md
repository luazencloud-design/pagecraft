# 과금/배포 모델 (현재 구현)

> 이전 "크레딧 유료화 계획서"는 폐기됨. 실제 구현된 모델은 아래 **BYOK + 초대 체험**.

---

## 현재 모델: 두 경로 공존

| 경로 | 누가 비용 | 통제 |
|---|---|---|
| **BYOK** (본인 Gemini 키) | 사용자가 Gemini에 직접 지불 | 통제 불필요(무제한 열어도 우리 비용 0) |
| **무료 체험** (초대 링크) | 운영자(서버 키) | 초대 링크로 입장 제한 + 30일/500크레딧 |

- 신규 사용자는 **관리자 발급 초대 링크**로만 무료 체험 진입 → "내가 보낸 사람만" 통제, 과거 사용자 자동 배제.
- 체험 끝나면 자연스럽게 **BYOK로 전환** 유도 (안내 문구 내장).

## 비용 구조 (무료 체험)

- 크레딧: `generate 1 / image 5 / bg-remove 5 / regen 1 / gift 1`, 1인 500개·30일
- 대략 한 체험 계정 최대 비용 ≈ 이미지 100장 × $0.04 ≈ $4 수준 (전부 이미지에 쓸 경우)
- **AI Studio 결제 상한**이 최종 안전장치

## 향후 유료화로 갈 경우 (아이디어)

- 초대별 크레딧/기간 커스텀 → "유료 플랜 = 큰 크레딧 초대" 식으로 수동 운영 가능
- 결제 연동(토스/스트라이프) + 크레딧 자동 충전은 미구현. 필요 시 `lib/trial.ts` 확장 지점.

## 관련 코드

- 크레딧/체험: `lib/trial.ts`
- 초대: `lib/invites.ts` + `/admin`
- 인가 분기: `lib/aiGate.ts`
