# 크리티컬 이슈 & 기술 부채

## 크리티컬 이슈

### 1. Vercel 서버리스 타임아웃

**문제:** 상세페이지 PNG 렌더링(`/api/render`)이 10~60초 소요.
Vercel 무료 플랜은 10초, Pro 플랜은 60초 제한.

**영향:** 이미지가 많거나 복잡한 레이아웃에서 렌더링 타임아웃 발생 가능.

**현재 대응:**
- Vercel Pro 사용 시 60초까지 가능
- 클라이언트에서 fallback HTML 미리보기 제공

**근본 해결:**
- 렌더링 서비스를 Railway/Render로 분리 (타임아웃 무제한)
- 또는 작업 큐(Bull Queue) 도입하여 비동기 처리

---

### 2. AI 이미지 생성 비용

**문제:** GPT Image 1.5가 $0.04/장. 상품 1개당 7장 = $0.28.
사용자 재생성까지 고려하면 건당 $0.35~$0.50.

**영향:** 사용량 급증 시 월 비용 예측 어려움.

**대응:**
- 사용자별 일일/월간 생성 한도 설정 필요
- 벌크 작업은 Imagen 4 Fast($0.02/장)로 분기
- 생성 전 미리보기로 불필요한 재생성 최소화

---

### 3. 쿠팡 데이터 수집 법적 리스크

**문제:** 쿠팡 자동완성/검색결과 스크래핑은 이용약관 위반 가능.

**영향:** 서비스 차단, 법적 분쟁 가능.

**대응:**
- 쿠팡 파트너스 API 범위 내 데이터만 사용 (우선)
- 스크래핑은 공개 데이터에 한정, 최소 빈도로 제한
- User-Agent/IP 로테이션 미사용 (공격적 크롤링 회피)
- 법적 검토 후 운영 판단 필요

---

### 4. 한글 폰트 렌더링

**문제:** 서버사이드 Canvas 렌더링 시 한글 폰트가 로딩되지 않으면 깨짐.

**영향:** 상세페이지 PNG에 한글이 네모(□)로 표시.

**현재 대응:**
- `public/fonts/`에 NotoSansKR 포함
- 빌드 시 폰트 다운로드 스크립트 (`setup-fonts.js`)
- CDN 다운로드 → /tmp 캐시 → 번들 폰트 fallback 체인

**주의:** Vercel 서버리스 환경에서는 `/tmp`이 함수 인스턴스 간 공유되지 않음.
`fonts/`를 `vercel.json`의 `includeFiles`에 포함해야 함.

## 기술 부채

### 기존 코드(pagecraft-pro)에서 이식 시 주의점

| 항목 | 문제 | 이식 시 개선 |
|------|------|-------------|
| `render.js` (469줄) | 단일 함수, 좌표 하드코딩 | `templates/`로 분리, 레이아웃 JSON 설정화 |
| AI 프롬프트 | HTML 인라인 문자열 | `services/ai.service.ts`에 프롬프트 빌더로 분리 |
| 월간 제한 카운터 | 인메모리 (서버리스 리셋) | Supabase DB 또는 Redis로 영구 저장 |
| 이미지 Base64 전송 | 대용량 body (10장 x 800px) | Supabase Storage에 업로드 후 URL로 참조 |
| 에러 처리 | console.error 후 무시 | 글로벌 에러 핸들러 + 사용자 피드백 UI |
